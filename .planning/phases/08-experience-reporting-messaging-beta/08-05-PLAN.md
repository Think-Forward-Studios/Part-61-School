    **docs/onboarding-runbook.md** — Step-by-step procedure. Structure:

    ```markdown
    # Partner-School Onboarding Runbook

    **Audience:** Part 61 School operator, flight school owner, chief flight instructor.
    **Goal:** Stand up a working instance of the app for a new school on a fresh Supabase project.
    **Time estimate:** 2–3 hours first time; ~30 minutes subsequent schools.

    ## Prerequisites

    - A Supabase account (https://supabase.com) — free tier is sufficient for beta
    - A Resend account (https://resend.com) — free tier (3,000/mo, 100/day)
    - A domain for email sending (e.g. school.example.com) — DKIM + SPF configuration access at your DNS registrar
    - GitHub account + access to the Part 61 School repo
    - Node 22 + pnpm 9

    ## Part A — Supabase project setup

    1. Create a new Supabase project at https://supabase.com/dashboard/new.
    2. In the project's **Settings > Database**, copy:
       - Connection string (transaction pooler, port 6543) → `DATABASE_URL`
       - Direct connection string (port 5432) → `DIRECT_DATABASE_URL`
    3. In **Settings > API**, copy:
       - Project URL → `NEXT_PUBLIC_SUPABASE_URL`
       - anon public key → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
       - service_role key → `SUPABASE_SERVICE_ROLE_KEY` (keep secret)
    4. In **Database > Extensions**, enable: `pg_cron`, `pg_net`, `btree_gist`, `uuid-ossp`, `postgis` (if using geofence).
    5. In **Authentication > URL Configuration**, set Site URL = your deploy URL. Add additional redirect URLs for local dev.
    6. In **Database > Replication**, confirm that the `supabase_realtime` publication exists (default).

    ## Part B — Resend email setup

    1. Sign up at https://resend.com.
    2. In **Domains**, add your sending domain (e.g. `school.example.com`).
    3. Copy the DKIM + SPF DNS records to your domain registrar.
    4. Wait ~24h (usually minutes) for verification.
    5. In **API Keys**, create a new key → `RESEND_API_KEY`.
    6. Set `RESEND_FROM_EMAIL=noreply@school.example.com`.

    ## Part C — Application deploy

    1. Clone the repo; `pnpm install`.
    2. Copy `.env.example` → `.env.local`; fill in all values from Parts A + B + a random 64-char `INTERNAL_WORKER_SECRET`.
    3. Run migrations: `pnpm --filter @part61/db migrate`.
    4. Seed the AC 61-65 endorsement catalog + 3 system courses: `pnpm --filter @part61/db seed` (if a seed command exists; otherwise the system courses are already in migrations 0019–0020).
    5. Configure pg_cron secrets: `psql "$DATABASE_URL" -c "select set_config('app.internal_worker_secret', '<your-secret>', false);"`
    6. Deploy the web app (Vercel recommended). Set all env vars in the deploy environment.

    ## Part D — First admin user

    1. Sign up via `/register` (self-registration may be gated — use the admin-invite flow).
    2. Or use Supabase Auth UI to create the admin user manually + run `insert into public.users (id, email, status, school_id) values (...)` with service_role.
    ```
