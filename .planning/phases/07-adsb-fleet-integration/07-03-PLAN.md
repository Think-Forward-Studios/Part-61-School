    **1. Replay page shell** (`fleet-map/replay/[tailNumber]/page.tsx`):
    Server component using `next/dynamic` with `{ ssr: false }` to import ReplayClient. Extract `tailNumber` from the route params. Pass it as a prop. Dark loading placeholder.

    **2. ReplayClient** (`fleet-map/replay/[tailNumber]/ReplayClient.tsx`):
    `'use client'` component:
    - Fetch track data: `trpc.adsb.flightTrack.useQuery({ tailNumber, minutes: 120 })`.
    - If no track found: show "No recent flight data for {tailNumber}" message with a "Back to Fleet Map" link.
    - Build a GeoJSON LineString from `track.lons` and `track.lats` (zip into coordinate pairs `[lon, lat]`). Include altitude as a third coordinate if available: `[lon, lat, alt]`.
    - Render MapLibre map with dark-matter style (same as fleet map).
    - Fit map bounds to the track extent using `map.fitBounds()` with padding.
    - Render `<ReplayTrackLayer>` with the LineString.
    - Render `<PlannedRouteOverlay>` if `plannedRoute` is non-null.
    - Render `<ReplayControls>` for playback.
    - **Animation:** Use `requestAnimationFrame` to animate an aircraft icon along the track coordinates. Maintain a `progress` state (0.0 to 1.0) that advances based on playback speed. At each frame, interpolate position along the LineString and update a point source for the aircraft marker. The aircraft icon rotates to match the heading between consecutive track points (compute bearing from consecutive lat/lon pairs).
    - Load the same SDF airplane icon as the fleet map (`/icons/airplane-sdf.png`).
    - Navigation: "Back to Fleet Map" link at top. Tail number in header.

    **3. ReplayTrackLayer** (`_components/ReplayTrackLayer.tsx`):
    Renders the flight track as a graduated polyline:
    - GeoJSON source with `lineMetrics={true}` (CRITICAL -- line-gradient won't work without this, per Pitfall 5).
    - Line layer with `line-gradient` paint property using `line-progress`:
      - 0.0 -> `#3b82f6` (blue, oldest point)
      - 0.5 -> `#22c55e` (green, middle)
      - 1.0 -> `#ef4444` (red, newest point)
    - `line-width: 4`, `line-cap: 'round'`, `line-join: 'round'`.
    - Follow Pattern 5 from 07-RESEARCH.md exactly.

    **4. PlannedRouteOverlay** (`_components/PlannedRouteOverlay.tsx`):
    If the aircraft has an active XC reservation with `route_string`:
    - Parse the route string. Route strings from Phase 3 are airport/waypoint identifiers separated by spaces or dashes (e.g., "KDFW KACT KAUS"). This component renders them as a dashed line.
    - For v1: since we don't have a waypoint geocoding service wired up, render the planned route as a text overlay or, if airports are available from the ADS-B Tracker's `/api/airports` endpoint, attempt to geocode them. If geocoding is not feasible, show the route string as a text label on the map header: "Planned route: KDFW -> KACT -> KAUS". The visual diff between planned and actual is already visible from the track itself.
    - If geocoded coordinates are available, render as a dashed line layer: `line-dasharray: [4, 4]`, `line-color: '#a78bfa'` (light purple), `line-width: 2`, `line-opacity: 0.7`.

    **5. ReplayControls** (`_components/ReplayControls.tsx`):
    Floating control panel at the bottom of the replay map:
    - **Slider:** range input from 0 to 100 (maps to 0.0-1.0 progress). Draggable. Updates aircraft position on the track.
    - **Play/Pause button:** toggles animation.
    - **Speed selector:** 1x / 2x / 4x buttons (default 2x per research recommendation). Speed multiplies the base animation rate.
    - **Flight info:** show departure time (firstSeen), current time (interpolated), total duration.
    - **Altitude readout:** show current altitude at the slider position.
    - Styled consistently with the dark map theme -- semi-transparent dark background, white text.

    **6. Dispatch deep-links** (`apps/web/app/(app)/dispatch/DispatchBoard.tsx`):
    Modify the existing DispatchBoard component:
    - For each row in the "Currently Flying" and "About to Fly" panels, add a small map icon/link that navigates to `/fleet-map?center={tailNumber}` (or just `/fleet-map` -- the fleet map will show the aircraft). The link should be a small airplane/map icon next to the aircraft tail number.
    - Use a simple `<a href="/fleet-map">` link with a map pin or radar icon. Keep it minimal -- the dispatch screen is dense.
    - For "Currently Flying" rows, the link text/tooltip should say "Track on Fleet Map".

    **7. Aircraft popup replay link** (`_components/AircraftPopup.tsx`):
    Update the existing AircraftPopup from Plan 02:
    - Add a "View last flight" link at the bottom of the popup that navigates to `/fleet-map/replay/{tailNumber}`.
    - This sits alongside the existing "View Aircraft" deep-link to /admin/aircraft/[id].
