    **4. AircraftLayer** (`_components/AircraftLayer.tsx`):
    Renders school fleet as a MapLibre GeoJSON source + symbol layer:
    - Source: GeoJSON FeatureCollection built from fleet positions. Each feature has properties: `tailNumber`, `heading` (trueTrack), `status` ('airborne' | 'ground' | 'grounded' | 'no_signal' | 'dispatched'), `staleSeconds` (computed from `Date.now()/1000 - apiTime`), `aircraftId`.
    - Status derivation: if `isGrounded` -> 'grounded'; else if `staleSeconds > 300` -> 'no_signal'; else if `staleSeconds > 60` -> 'stale'; else if `onGround` -> 'ground'; else -> 'airborne'. If aircraft is dispatched but has no position yet -> 'dispatched'.
    - Symbol layer spec: Follow Pattern 2 from 07-RESEARCH.md exactly. `icon-image: 'airplane'` (SDF), `icon-size: 0.8`, `icon-rotate: ['get', 'heading']`, `icon-rotation-alignment: 'map'`, `icon-allow-overlap: true`. Data-driven `icon-color`: green (#22c55e) for airborne, yellow (#eab308) for ground, red (#ef4444) for grounded, blue (#3b82f6) for dispatched, gray (#6b7280) for no_signal. Data-driven `icon-opacity`: 0.3 for staleSeconds>300, 0.6 for staleSeconds>60, 1.0 otherwise.
    - Text label: `text-field: ['get', 'tailNumber']`, white with black halo.
    - **Signal-lost pulsing:** For aircraft with `staleSeconds > 300`, use a `requestAnimationFrame` loop that toggles the layer's `icon-opacity` filter for those features between 0.2 and 0.5 every 500ms. Alternatively, since this is <5 aircraft, overlay HTML markers via `Marker` component from react-map-gl with a CSS `@keyframes pulse` animation for just the signal-lost aircraft.
    - Click handler: on click of the fleet layer, read `aircraftId` from clicked feature properties, call `router.push('/admin/aircraft/${aircraftId}')` for deep-link (ADS-04). Show AircraftPopup on hover or click before navigation.

    **1. useGeofenceCheck hook** (`_components/useGeofenceCheck.ts`):
    Custom hook that runs the client-side geofence check on each poll cycle:
    - Takes: fleet positions array, active geofence (from `trpc.admin.geofence.getActive` query).
    - Uses `@turf/boolean-point-in-polygon` and `@turf/circle` per Pattern 6 in 07-RESEARCH.md.
    - For circle geofences: convert center + radiusNm to a 72-sided polygon using `turfCircle(center, radiusNm * 1.852, { steps: 72 })`.
    - For each school aircraft position, check if inside the geofence polygon.
    - Returns `{ outsideAircraft: { tailNumber, latitude, longitude, heading }[] }`.
    - Only checks airborne/ground aircraft (skip grounded -- they have no real position).
    - Memoize the geofence polygon conversion so it doesn't recompute on every 5s cycle (only when geofence changes).

    **2. GeofenceOverlay** (`_components/GeofenceOverlay.tsx`):
    Renders the active geofence boundary on the map:
    - Fetches geofence via `trpc.admin.geofence.getActive.useQuery()`.
    - If geofence exists and is a circle, convert to polygon via `turfCircle` for rendering.
    - Render as a MapLibre fill layer (semi-transparent blue: `fill-color: '#3b82f6'`, `fill-opacity: 0.08`) plus a line layer (border: `line-color: '#3b82f6'`, `line-width: 2`, `line-opacity: 0.5`). Subtle per user decision -- "not a thick border".
    - If no geofence, render nothing.

    **3. GeofenceEditor** (`_components/GeofenceEditor.tsx`):
    Admin-only geofence drawing tool:
    - Only rendered when the current user has admin role (check from session/context).
    - Uses `terra-draw` with `@watergis/maplibre-gl-terradraw` as a MapLibre IControl.
    - Access the map instance via `useMap()` hook. Add/remove the terra-draw control in a `useEffect` cleanup.
    - Two modes: polygon draw and circle draw. Toggle buttons in a floating panel.
    - On draw finish (terra-draw `finish` event): extract the GeoJSON geometry from terra-draw. For circle mode, extract center point + compute radius from drawn circle. Call `trpc.admin.geofence.upsert.useMutation()` with `{ kind, geometry, radiusNm, label }`.
    - Show save/cancel buttons after drawing. On save: call the mutation. On cancel: clear the drawn shape.
    - After successful save: invalidate the geofence query to refresh the overlay. Show a brief success toast or inline confirmation.
    - Include a "Delete geofence" button that calls `admin.geofence.softDelete`.

    **4. GeofenceAlert** (`_components/GeofenceAlert.tsx`):
    Alert banner for out-of-geofence aircraft:
    - Consumes the `outsideAircraft` array from `useGeofenceCheck`.
    - If any aircraft are outside: render a red banner at the top of the map viewport (below the feed status banner if present). Content: "Warning: {tailNumber} outside training area -- last seen at {lat}, {lon} heading {hdg} degrees". If multiple aircraft outside, list all.
    - Auto-clear: when the array becomes empty on next poll, banner disappears automatically.
    - Per user decision: red banner, not just yellow. The banner should feel urgent.

    **5. Wire into FleetMapClient** (`FleetMapClient.tsx`):
    Update FleetMapClient to:
    - Include `<GeofenceOverlay />`, `<GeofenceEditor />`, `<GeofenceAlert outsideAircraft={...} />`.
    - Call `useGeofenceCheck(fleetPositions, activeGeofence)` and pass results to GeofenceAlert.
    - Conditionally render GeofenceEditor only for admin role.
