<objective>
Build the entire Postgres foundation: Drizzle schema (schools, bases, users, user_roles, documents, audit_log), the role + mechanic_authority enums (AUTH-06, AUTH-07), the generic audit trigger function and append-only audit_log (FND-03), the hard-delete-block trigger (FND-04), Supabase's custom access token hook that injects school_id/roles/active_role into the JWT (FND-01 wiring for RLS), RLS policies keyed off auth.jwt() on every business table (FND-01, FND-02), and a Vitest cross-tenant RLS harness that runs against a locally-started Supabase stack and asserts a school A user cannot read or write school B rows (FND-01 verification gate). Timezone discipline (FND-06) is enforced by using `timestamp({ withTimezone: true })` everywhere and storing school/user IANA timezone names.

Purpose: This is the "cannot be retrofitted later" core. Every later phase's schema additions will follow the same trigger/policy/test-registration template established here.
Output: A migration that, when applied to a fresh Postgres, produces the full Phase 1 data model with working RLS and audit; and a green Vitest harness that proves cross-tenant isolation.
</objective>
