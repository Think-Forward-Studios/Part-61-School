<objective>
Add document storage (FND-07): Supabase Storage `documents` bucket, RLS policies keyed on the school_<id>/user_<id>/ path prefix, a tRPC documents router with server-issued signed upload URLs + server-mediated signed download URLs, and a minimal /profile/documents page where a user can upload a medical/license/ID/insurance file and see their uploaded documents. Uploads are authenticated end-to-end: the server validates MIME + size + permissions before issuing the signed upload URL, and downloads flow through a tRPC procedure that performs the Drizzle SELECT under RLS before calling createSignedUrl. Every insert/update on the documents table fires the audit trigger from plan 02.

    import { z } from 'zod';
    export const DocumentKind = z.enum(['medical', 'pilot_license', 'government_id', 'insurance']);
    export type DocumentKind = z.infer<typeof DocumentKind>;
    export const ALLOWED_MIME_TYPES = ['image/jpeg', 'image/png', 'application/pdf'] as const;
    export const MAX_BYTE_SIZE = 25 * 1024 * 1024; // 25 MB
    export function extForMime(mime: string): string {
      return { 'image/jpeg': 'jpg', 'image/png': 'png', 'application/pdf': 'pdf' }[mime] ?? 'bin';
    }
    export function storagePath(schoolId: string, userId: string, documentId: string, ext: string): string {
      return `school_${schoolId}/user_${userId}/${documentId}.${ext}`;
    }

    - Imports `documents` schema, `documentKindEnum` from @part61/db; `DocumentKind`, `ALLOWED_MIME_TYPES`, `MAX_BYTE_SIZE`, `storagePath`, `extForMime` from @part61/domain
    - Service-role supabase-js client created lazily (same pattern as auth.ts from plan 03)
    - `createSignedUploadUrl` (protectedProcedure):
      - input zod: { kind: DocumentKind, mimeType: enum(...ALLOWED_MIME_TYPES), byteSize: number().int().positive().max(MAX_BYTE_SIZE), expiresAt: date().optional() }
      - Generate a documentId via crypto.randomUUID()
      - Compute path = storagePath(ctx.session.schoolId, ctx.session.userId, documentId, extForMime(mimeType))
      - Call `supabase.storage.from('documents').createSignedUploadUrl(path)` with the service-role client
      - Return { documentId, path, signedUrl, token }
      - Do NOT insert a documents row yet (wait for finalize)
    - `finalizeUpload` (protectedProcedure):
      - input: { documentId: uuid, kind: DocumentKind, path: string, mimeType: string, byteSize: number, expiresAt?: Date }
      - Validation: path must equal storagePath(ctx.session.schoolId, ctx.session.userId, documentId, ext) — reject otherwise (defense against path tampering)
      - (Optional) Call `supabase.storage.from('documents').list(...)` or `getPublicUrl(path)` HEAD to confirm the object exists before inserting the row
      - Insert via ctx.tx into documents (id=documentId, schoolId=session.schoolId, userId=session.userId, kind, storagePath=path, mimeType, byteSize, expiresAt, uploadedBy=session.userId)
      - Returns the inserted row
    - `list` (protectedProcedure):
      - Lists ctx.session.userId's documents where deletedAt IS NULL, ordered by uploadedAt DESC
      - RLS + withSchoolContext ensures only own-school rows are visible
    - `createSignedDownloadUrl` (protectedProcedure):
      - input: { documentId: uuid }
      - SELECT the row via ctx.tx (RLS scoped). If not found → NOT_FOUND.
      - Call `supabase.storage.from('documents').createSignedUrl(row.storagePath, 300)` — 5 minute TTL
      - Return { url }
    - `softDelete` (protectedProcedure):
      - input: { documentId: uuid }
      - UPDATE ctx.tx documents set deletedAt=now() where id=documentId. RLS ensures own-school only.
      - Returns void. Audit trigger captures the soft_delete action automatically.

AVOID: Using `createSignedUrl` for uploads — that's a read URL. Use `createSignedUploadUrl` specifically.
